# RL 训练指标解读指南

本文档总结了 RL（强化学习）训练中各项指标的含义和判断方法，帮助快速诊断训练状态。

## 二、核心效果指标

### 2.1 验证集 Reward / Accuracy

| 指标 | 含义 | 判断方法 |
|------|------|---------|
| `val-*/reward/mean@1` | 验证集上生成 1 次的平均 reward | **越高越好**，看趋势是否上升 |
| `val-*/acc/mean@1` | 验证集准确率 | **越高越好**，最直观的效果指标 |

**示例分析**：
```
初始: val-acc/mean@1 = 0.386
训练后: val-acc/mean@1 = 0.452
结论: 准确率提升 17%，训练有效
```

## 三、训练稳定性指标

### 3.1 KL Loss (KL 散度损失)

#### 直观理解

```
┌─────────────────────────────────────────────────────────────────┐
│                      KL Loss 的含义                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Reference Model (冻结)     Actor Model (训练中)               │
│         π_ref                    π_θ                           │
│           │                        │                           │
│           │    输入同一个 prompt   │                           │
│           │    ─────────────────►  │                           │
│           │                        │                           │
│        输出概率分布            输出概率分布                     │
│        [0.1, 0.3, 0.5, 0.1]  [0.2, 0.2, 0.4, 0.2]            │
│                ↘                ↙                              │
│                   KL 散度                                       │
│                = 两个分布的差异                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 为什么需要 KL Loss？

```
┌─────────────────────────────────────────────────────────────────┐
│  没有 KL 约束会怎样？                                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ❌ 没有 KL 约束:                                               │
│     Actor 可能学会: "不管题目问什么，我都输出最常见的答案"       │
│     → 短期 reward 可能不错                                      │
│     → 但丢失了原有的语言能力、推理能力                          │
│     → 这叫 "Reward Hacking" 或 "策略退化"                       │
│                                                                 │
│  ✅ 有 KL 约束:                                                 │
│     Actor 必须在 "提升 reward" 和 "保持原有能力" 之间平衡       │
│     → 保留了预训练模型的知识                                    │
│     → 同时学会解决新任务                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 判断标准

| KL Loss 值 | 含义 | 状态 |
|-----------|------|------|
| < 0.01 | Actor 几乎没变 | 可能学习太慢 |
| 0.01 - 0.1 | Actor 在温和地改变 | ✅ 理想状态 |
| 0.1 - 0.5 | Actor 偏离较多 | ⚠️ 需要关注 |
| > 0.5 | Actor 改变剧烈 | ⚠️ 可能丢失原有能力 |
| > 1.0 | 严重偏离 | ❌ 可能模型崩溃风险 |

#### 相关配置

```yaml
actor_rollout_ref.actor.use_kl_loss=True      # 启用 KL 损失
actor_rollout_ref.actor.kl_loss_coef=0.01     # KL 损失系数
actor_rollout_ref.actor.kl_loss_type=low_var_kl  # KL 计算方式
```

**调优建议**：
- KL 持续升高 → 增加 `kl_loss_coef`
- KL 始终为 0 或很低 → 可适当降低 `kl_loss_coef`

### 3.2 Entropy (策略熵)

#### 直观理解

```
┌─────────────────────────────────────────────────────────────────┐
│                      Entropy 的含义                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  Entropy = 策略的 "不确定性" 或 "多样性"                        │
│                                                                 │
│  高熵: 选词不确定，输出多样                                     │
│  ┌─────────────────────────────────┐                           │
│  │ 概率分布: [0.25, 0.25, 0.25, 0.25]                          │
│  │ 熵值: 高                        │                           │
│  │ 含义: 模型不知道选哪个更好      │                           │
│  │       所有选项都有可能          │                           │
│  └─────────────────────────────────┘                           │
│                                                                 │
│  低熵: 选词很确定，输出固定                                     │
│  ┌─────────────────────────────────┐                           │
│  │ 概率分布: [0.97, 0.01, 0.01, 0.01]                          │
│  │ 熵值: 低                        │                           │
│  │ 含义: 模型非常确定选第一个      │                           │
│  │       几乎没有探索空间          │                           │
│  └─────────────────────────────────┘                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 为什么 Entropy 重要？

```
┌─────────────────────────────────────────────────────────────────┐
│  Entropy 太低会怎样？—— 策略坍塌        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  正常训练:                                                       │
│  Prompt: "这道题的答案是?"                                      │
│  Output: "根据图像分析，角度约为 45°..." (多样、自然)          │
│                                                                 │
│  策略坍塌后:                                                     │
│  Prompt: "这道题的答案是?"                                      │
│  Output: "45°"  (永远只输出这个，entropy ≈ 0)                  │
│  Prompt: "请详细分析图像"                                       │
│  Output: "45°"  (不管问什么，都输出这个)                        │
│                                                                 │
│  为什么会坍塌？                                                  │
│  → RL 学到了 "45° 这个答案经常对"                               │
│  → 于是把 45° 的概率推到 99%                                    │
│  → 不再探索其他可能                                              │
│  → 最终变成一个只会输出 "45°" 的笨模型                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 判断标准

| Entropy 值 | 含义 | 状态 |
|-----------|------|------|
| > 1.0 | 策略很不确定，探索性强 | 可能学不到东西 |
| 0.2 - 0.5 | 有一定确定性，但保持探索 | ✅ 理想状态 |
| 0.1 - 0.2 | 开始变得过于确定 | ⚠️ 需要关注 |
| < 0.1 | 策略过于确定 | ❌ 可能策略坍塌 |
| ≈ 0 | 策略坍塌 | ❌ 模型废了 |

#### 相关配置

```yaml
actor_rollout_ref.actor.entropy_coeff=0  # 熵奖励系数，可设为 0.01 鼓励探索
```

**调优建议**：
- Entropy 快速下降 → 设置 `entropy_coeff=0.01` 鼓励探索
- Entropy 始终很高不收敛 → 可考虑降低 `entropy_coeff`

---

### 3.3 PPO Clip Fraction (pg_clipfrac)

#### 含义

PPO 使用 clip 来限制策略更新的幅度。`clipfrac` 表示有多少比例的 token 被 clip 了。

```
ratio = π_θ(a|s) / π_old(a|s)  # 新旧策略概率比

if ratio > 1 + ε or ratio < 1 - ε:
    → 被 clip 了
```

#### 判断标准

| clipfrac | 含义 | 状态 |
|----------|------|------|
| ≈ 0 | 几乎没被 clip | 新策略和旧策略几乎一样，可能学太慢 |
| 0.01 - 0.1 | 适度 clip | ✅ 正常学习 |
| > 0.1 | 大量被 clip | 策略更新太激进 |

#### 相关配置

```yaml
actor_rollout_ref.actor.clip_ratio=0.2  # PPO clip 范围 (1-ε, 1+ε)
```

---

### 3.4 Gradient Norm (grad_norm)

#### 含义

梯度范数反映梯度的大小。太大可能训练不稳定，太小可能学不动。

#### 判断标准

| grad_norm | 状态 |
|-----------|------|
| 1 - 100 | ✅ 正常范围 |
| > 1000 | ⚠️ 可能梯度爆炸 |
| < 0.001 | ⚠️ 可能梯度消失 |

#### 相关配置

```yaml
actor_rollout_ref.actor.optim.clip_grad=1.0  # 梯度裁剪阈值
```

## 四、KL Loss vs Entropy 关系

```
┌─────────────────────────────────────────────────────────────────┐
│                    KL Loss vs Entropy                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  KL Loss:                                                       │
│  ├── 衡量: 当前策略 vs 初始策略的差异                           │
│  ├── 目的: 防止模型偏离太远，保留预训练知识                     │
│  └── 太高 → 丢失原有能力                                        │
│                                                                 │
│  Entropy:                                                       │
│  ├── 衡量: 当前策略本身的不确定性                               │
│  ├── 目的: 保持探索能力，防止策略坍塌                           │
│  └── 太低 → 策略坍塌，输出单一                                  │
│                                                                 │
│  共同作用:                                                       │
│  ├── KL 限制 "改变幅度"                                         │
│  ├── Entropy 限制 "确定性程度"                                  │
│  └── 两者共同保证训练稳定                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**类比理解**：

| 指标 | 类比 | 太高的问题 | 太低的问题 |
|------|------|-----------|-----------|
| **KL Loss** | 车速 | 偏离太多，丢失原有能力 | 学太慢 |
| **Entropy** | 方向盘灵活性 | 学不确定，难收敛 | 策略坍塌，只会一种输出 |

## 五、训练诊断速查表

### 5.1 快速判断训练是否成功

| 问题 | 看什么指标 | 健康状态 |
|------|-----------|---------|
| **模型变强了吗？** | `val-*/acc/mean@1` | 持续上升 |
| **训练稳定吗？** | `kl_loss`, `entropy`, `grad_norm` | 在合理范围内 |
| **策略坍塌了吗？** | `entropy` | 不持续下降到接近 0 |
| **学太快/太慢？** | `pg_clipfrac` | 0.01 - 0.1 |

### 5.2 常见问题诊断

| 症状 | 可能原因 | 解决方案 |
|------|---------|---------|
| KL 持续升高 | 策略偏离太快 | 增加 `kl_loss_coef` |
| Entropy 快速下降到 0 | 策略坍塌 | 增加 `entropy_coeff` |
| clipfrac 接近 0 | 学习太慢 | 检查 reward 设计，或增大学习率 |
| clipfrac > 0.1 | 更新太激进 | 降低学习率 |
| grad_norm 爆炸 | 梯度不稳定 | 降低 `clip_grad` 阈值 |
| val acc 不提升 | reward 设计问题或任务太难 | 检查 reward 函数 |

## 六、指标监控最佳实践

### 6.1 训练时重点关注

```
必须监控 (红色警报指标):
├── val-*/acc/mean@1     → 效果有没有提升
└── actor/entropy        → 是否策略坍塌

建议监控 (黄色警报指标):
├── actor/kl_loss        → 是否偏离太多
├── actor/pg_clipfrac    → 学习是否正常
└── actor/grad_norm      → 梯度是否健康

可选监控 (性能优化):
├── timing_s/step        → 训练速度
└── perf/max_memory_*    → 内存使用
```

### 6.2 正常训练曲线特征

```
✅ 正常训练:

  val_acc  ────────────────────────► (上升后趋于平稳)
              ___________
             /
            /
  ─────────

  kl_loss  ────────────────────────► (先上升后稳定)
                 ________
                /
               /
  ───────────

  entropy  ────────────────────────► (缓慢下降后稳定)
                _______________
               /
              /
  ──────────

❌ 异常训练:

  entropy (策略坍塌) ─────────────►
              \
               \
                \_______________ (快速下降到 0)

  kl_loss (偏离太快) ─────────────►
                              /
                             /
                            / (持续上升不收敛)
```

## 七、实际案例：Qwen2.5-VL GRPO 训练

### 训练配置

```yaml
algorithm.adv_estimator=grpo
actor_rollout_ref.actor.use_kl_loss=True
actor_rollout_ref.actor.kl_loss_coef=0.01
actor_rollout_ref.actor.entropy_coeff=0
```

### 训练过程指标

| Step | val_acc | kl_loss | entropy | 状态 |
|------|---------|---------|---------|------|
| 0 | 0.386 | - | - | 初始 |
| 55 | 0.452 | 1.22 | 0.259 | 有效但 KL 稍高 |

### 诊断结论

```
✅ 好的信号:
├── 验证准确率从 0.386 → 0.452 (提升 17%)
├── entropy = 0.259 (策略没有坍塌)
├── pg_clipfrac = 0.013 (正常学习)
└── grad_norm = 14.27 (梯度正常)

⚠️ 需要关注:
├── kl_loss = 1.22 (稍高，模型改变较大)
└── 如果 KL 继续升高，考虑增加 kl_loss_coef

建议:
└── 继续训练，观察 KL 是否稳定或继续上升
```

## 八、总结

判断 RL 训练效果，核心关注：

1. **`val-*/acc/mean@1`** → 效果有没有提升 (最核心)
2. **`entropy`** → 是否策略坍塌 (防止模型废掉)
3. **`kl_loss`** → 是否偏离太多 (保持原有能力)

牢记一句话：**RL 训练的目标是在提升任务效果的同时，保持模型原有能力不丢失。**
